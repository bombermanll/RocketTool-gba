# 火箭队修改工具（RocketTool）

面向 GBA 改版《西班牙火箭队一至五周目汉化版》的实时修改与存档编辑工具。

程序使用 C#、.NET 和 Avalonia 构建。每次启动先明确选择游戏版本，再选择连接 mGBA 修改当前游戏内存，或直接读取 `.sav` / `.srm` 文件进行安全保存。

> 当前主要适配“汉化 by 兔砸博士 v2.1”及本项目用于逆向验证的同版 ROM。宝可梦、道具、招式和存档结构都包含改版专属内容，不保证兼容其他 ROM、其他汉化版本或原版游戏。

## 两种编辑模式

| 模式 | 用途 | 写入方式 | 当前范围 |
| --- | --- | --- | --- |
| mGBA 实时模式 | 修改正在运行的游戏 | 通过 `mgba_bridge.lua` 直接读写模拟器内存 | 队伍、背包、箱子、训练家、图鉴导入和实验功能 |
| 存档模式 | 离线编辑 `.sav` / `.srm` | 修改内存中的副本，保存前创建一次性原始 `.bak` | 队伍、背包和箱子 |

切换到存档模式后，mGBA 会被视为断开。训练家信息、图鉴导入和实验功能不会在存档模式中开放，避免把尚未建立可靠存档映射的实时功能误用于文件写回。

## 主要功能

### 队伍与箱子编辑

- 以宝可梦图片显示队伍，并以接近游戏布局的 `6 x 5` 矩阵分箱显示 PC 宝可梦。
- 编辑种类、携带道具、性格、性别、特性槽、闪光状态、经验和亲密度。
- 编辑四个招式、当前 PP、PP 提升次数、个体值和努力值。
- 显示属性、总种族值和能力值预览。
- 显示并编辑初始训练家名字；实时模式可同步当前玩家名字。
- 识别蛋并显示 ROM 中的蛋图标，可编辑 `0..255` 的剩余孵化周期。
- 修改 Gen 3 宝可梦数据时重排加密子结构并重算单体 checksum。

### 背包编辑

- 按普通道具、回复药品、精灵球、战斗道具、树果、宝物、招式/秘传机器和重要物品分类。
- 实时模式支持读取、校准、覆盖当前格和添加道具。
- 存档模式可直接在左侧列表搜索并更换道具；数量在回车或失焦后自动更新待保存副本。
- 支持获取当前分类全部道具，以及分别获取全部技能机和全部秘传机。
- 技能机与秘传机使用从 ROM 确认的完整编号范围，并在存档中按道具编号排序。
- 批量写入数量默认设置为 `240`，降低背包满数量后无法拾取同类道具的风险。

重要物品可能与剧情标记和流程状态相关。批量获取重要物品可能造成剧情无法触发或进度卡住，不建议使用。

### 训练家信息

训练家页面仅用于 mGBA 实时模式：

- 读取和修改训练家名字。
- 显示 Trainer ID、Secret ID 和合并 OT ID。
- 读取和修改最高 `99,999,999` 的加密金钱字段。

训练家名字和 OT ID 使用 ROM 逆向确认的 SaveBlock2 字段。修改训练家身份后，已有宝可梦的初始训练家信息可能不再匹配，并可能出现不听话等游戏行为。

### 宝可梦图鉴

- 按中文名或编号搜索宝可梦。
- 显示图片、属性、总种族值、单项种族值和等级能力预览。
- 查看进化链和升级招式。
- 按总种族值或任一单项种族值升序、降序排列。
- 在实时模式中将选中的宝可梦导入队伍或箱子。

### 招式图鉴

- 显示改版 ROM 中的招式名称、属性、物理/特殊/变化分类、威力、命中和 PP。
- 支持名称搜索、属性筛选和招式分类筛选。

### 实验功能

实验页面仅用于 mGBA 实时模式，包括跨地图传送、无限 PP、锁血、关闭遇怪和穿墙等功能。这些功能直接影响游戏运行状态，部分功能仍具有实验性质，使用前必须创建 mGBA 即时存档。

## 快速开始

### 方式一：编辑存档文件

1. 启动 RocketTool，选择要修改的游戏版本，再切换到存档模式。
2. 点击顶部的 **选择存档**，选择 `.sav` 或 `.srm` 文件。
3. 在队伍、背包或箱子页面进行修改。
4. 确认模拟器没有正在写入该文件，点击顶部的 **保存修改并备份**。
5. 在 mGBA 中载入保存后的存档并检查结果；如需恢复，将 `.bak` 文件复制并改回原扩展名。

第一次保存前会在同目录创建 `<原文件名>.bak`，以后不会覆盖该备份。修改后的内容保存回原文件名；也可以使用 **另存为** 保留原存档不变。程序保留模拟器可能附加在 128 KiB 电池存档前后的数据，并要求当前存档存在一组可严格校验的 `14/14` section。只通过宽松规则读取的 section 会禁用写回。

保存过程会先写入同目录临时文件，再重新读取并检查 section、队伍、背包和箱子修改；备份及修改结果全部校验成功后才会替换原文件。如果原文件在读取后被模拟器或其他程序修改，工具会拒绝覆盖并要求重新读取。

### 方式二：连接 mGBA 实时修改

1. 使用 mGBA 加载对应 ROM 和游戏存档。
2. 在 mGBA 的脚本窗口加载 [mgba_bridge.lua](mgba_bridge.lua)。
3. 确认 mGBA 日志出现 bridge 已监听 `127.0.0.1:8765` 的提示。
4. 启动 RocketTool，点击顶部的 **连接 mGBA**。
5. 在对应页面读取数据、编辑并执行明确的写入操作。
6. 回到游戏确认修改，再使用游戏自身的存档功能保存。

实时模式会直接改变模拟器内存。修改前应先创建 mGBA 即时存档，背包和箱子列表与游戏不一致时应重新读取或校准，不能盲目写入。

## 运行与构建

### 从源码构建

要求安装 .NET SDK 10。主要开发环境为 macOS，Avalonia 项目也可面向 Windows 构建。

```sh
dotnet build RocketTool.Avalonia/RocketTool.Avalonia.csproj
```

构建命令行工具：

```sh
dotnet build RocketTool.Cli/RocketTool.Cli.csproj
```

## 数据安全与限制

- 无论使用哪种模式，都应保留未经修改的原始存档备份。
- 不要在游戏正在写入电池存档时替换 `.sav` / `.srm` 文件。
- 存档模式目前不修改训练家信息、金钱或实验功能相关状态。
- 实时模式中的写入立即生效，但仍需在游戏内正常保存才会进入电池存档。
- 重要物品、跨地图传送和实验辅助功能可能影响剧情或运行状态。
- 本工具的结构和偏移来自目标改版 ROM 与样本存档的逆向验证，不是通用 Gen 3 存档编辑器。

## 项目结构

```text
RocketTool.Avalonia/   桌面图形界面
RocketTool.Core/       Gen 3 数据、扫描、编辑与存档校验
RocketTool.Cli/        调试和逆向验证命令行工具
profiles/              按游戏版本保存 profile.json 与 ROM 派生 db
docs/                  界面图片和文档资源
mgba_bridge.lua        mGBA 实时通信脚本
```

## 游戏版本配置

启动图形界面后必须先选择要修改的游戏版本。程序不根据 ROM、存档或 GAMECODE 自动选择版本。

每个版本位于 `profiles/<profile-id>/`：

```text
profile.json           策略 ID、实时内存地址、ROM 表偏移、数据上限和功能开关
db/*.tsv               该版本的宝可梦、招式、道具、地图等派生数据
```

新增版本时应复制为新的 Profile，并只启用已经完成逆向和读写验证的功能。CLI 在只有一个 Profile 时自动使用它；安装多个 Profile 后可通过 `ROCKET_TOOL_PROFILE=<profile-id>` 明确选择。
